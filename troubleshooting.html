서비스를 운영하며 실제로 마주친 오류와 인프라 문제를 진단하고 해결한 기록입니다.

Case A

Postgres 드라이버 호환성 오류

문제

pgbouncer=true 쿼리 파라미터가 psycopg v3와 충돌하며 sqlalchemy.exc.ProgrammingError가 발생했습니다.

해결

해당 파라미터를 제거하고 postgresql+psycopg:// 드라이버로 전환, connect_args={"prepare_threshold": None}를 적용해 문제를 해결했습니다.

Case B

종합 버그 픽스 세션

발견된 이슈
  • 파티 종료 API 오류
  • 관리자 권한 대소문자 불일치
  • 누락된 모델 컬럼 · API 라우트
  • 탈퇴 기능 미완성 호출
  • 마이페이지 dynamic import 충돌
  • 챗봇 성능 저하 (LIMIT 200 적용)
결과

한 세션에서 6가지 이슈를 순차적으로 진단하고 수정하며 서비스 전반의 안정성을 확보했습니다. 증상 하나에 매몰되지 않고 전체 흐름을 점검하는 방식으로 접근했습니다.

Case C

인프라 마이그레이션 — Render → Oracle Cloud VPS

문제

Render 무료 티어의 배포 할당량이 소진되며 서비스 확장에 제약이 생겼습니다.

해결

Docker Compose로 백엔드와 DB를 컨테이너화해 Oracle Cloud VPS에 배포하고, Caddy 리버스 프록시와 iptables 방화벽 설정으로 Vercel 프론트엔드와의 HTTPS(mixed-content) 통신 문제까지 해결했습니다.
